# Code of Conduct

## Our Pledge

We as members of the Type or Die development team pledge to make participation in our project a respectful and collaborative experience for everyone. We are committed to providing a welcoming environment regardless of background, experience level, or personal identity.

## Our Standards

Examples of behavior that contributes to a positive environment:

- Communicating respectfully through Teams and GitHub
- Giving and accepting constructive feedback on pull requests
- Taking responsibility for mistakes and learning from them
- Supporting teammates who are stuck or need help

Examples of unacceptable behavior:

- Dismissing or ignoring a teammate's contributions
- Pushing code that intentionally breaks another teammate's work
- Taking credit for work done by someone else
- Ignoring agreed-upon deadlines without communicating

## Workflow Expectations

- All code changes go through pull requests before merging into main
- Team members review each other's PRs before merging
- Use the agreed-upon branch naming convention (feature/, ui/, asset/, fix/)
- Communicate blockers or delays through Microsoft Teams as soon as possible
- Test your changes locally in Godot before opening a PR

## Enforcement

If a team member is not meeting expectations, the issue will be raised in the team group chat first. If the behavior continues, it will be escalated to the course instructor.

## Scope

This code of conduct applies to all project spaces including the GitHub repository, Microsoft Teams group chat, and any in-person collaboration sessions.
